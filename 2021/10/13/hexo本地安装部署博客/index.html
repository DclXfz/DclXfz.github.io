使用hexo在本地安装部署博客流程

Created2021-10-13|Updated2021-10-16|博客搭建
|Post View:

前言

很久之前，我就萌生了想要自己写博客的想法，主要是为了记录自己的学习和生活。碍于能力有限，这个想法一直没有付诸行动，直到今天，我通过hexo和github搭建起了我自己博客的大体框架（这篇只写了本地部署，部署到github，gitee等在后续文章种进行）。虽然目前的博客还仅仅处于雏形，但是万事开头难，我的第一步已经迈出，相信以后的路会更容易。在搭建博客的过程中，我查阅了大量资料，有的有用，有的无用，为了防止未来做重复的工作，我决定记录这次搭建博客的全过程，一方面为自己，另一方面为别人，为那些与我志同道合的朋友，希望通过我的经历，可以帮助他们少走弯路，哪怕一点点。

准备工作

  1. Node.js
  2. git
  3. hexo
  4. github账号

(1)安装Node.js

Node.js下载网址：https://nodejs.org/zh-cn/download/

Windows直接点击红框下载，下载之后的结果如图：

安装过程一路点击next，安装路径可改可不改，如果不熟悉，直接next到最后即可。

安装后验证是否安装成功的标志：

用 Win + R 打开运行，输入 cmd 并进入cmd窗口，在cmd窗口中输入node -v和npm -v，如果安装正确会返回各自的版本。如下图：

(2)安装Git

官网下载速度超慢，直接用阿里云镜像下载，选择最新的版本：https://npm.taobao.org/mirrors/git-for-windows/

翻到页面底端，选择日期最新的版本，点进去后选择后缀为exe的可执行文件下载。选择哪个版本问题不大，效果是一样的。但是必须下啊在Git开头，exe结尾的文件，如下图我选择的版本。

下载后的结果如下图：

一路next安装到最后即可，可以选择修改安装路径。安装完后随便找一个文件夹，在空白处右键，菜单栏会多出两个选项Git GUI Here,和Git Bash Here

点击Git Bash Here，然后输入git –version可以查看安装好的git版本号，出现版本号，说明安装正确。正确的结果如下图：

至此Git安装完毕！

(3)安装hexo并生成demo博客，在本地预览效果

  1. 随便在电脑中选择一个目录用于存放博客内容。
  2. 在上面选中的目录下空白处右键点击Git Bash Here。
  3. 在打开的终端中输入npm install hexo-cli -g安装hexo。

  1. 在终端中输入hexo init demo_blob生成一个demo博客。

demo_blob是项目名字，同时在你选择的这个目录种会生成一个名为demo_blob的文件夹，该文件夹包含博客的基本内容。

在终端中进入这个文件夹：cd demo_blob

文件夹中的内容如下：

  1. 本地预览demo博客：hexo cl && hexo g && hexo s

  1. 在浏览器中输入终端最后一行的网址，最好鼠标选中，右键copy。回车之后浏览器就会显示demo博客：

当在终端输入ctrl + c后，预览结束，浏览器也无法显示，如果再想显示，需要再次输入：hexo cl && hexo g && hexo s。

总结：
开始本地预览输入：hexo cl && hexo g && hexo s
结束本地预览：ctrl + c
